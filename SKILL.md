## 核心功能

### 功能一：搜索示例启发

当用户需要某个类型的图片提示词时，先在 db 中搜索相关案例。

**执行步骤**：
1. 根据用户关键词，查「类型路由表」定位到对应类型目录
2. 读取 `db/<类型名>/prompt.md` 找到相关案例
3. 参考底部「输出模板 → 搜索案例输出」格式展示

### 功能二：模糊需求启发

当用户只有模糊描述（如"给我一个好看的图片"）时，使用启发式提问引导：

**启发框架**：
1. **明确类型** — 海报？界面？摄影？插画？
2. **锁定主体** — 要呈现什么？人物/产品/场景/抽象概念？
3. **确定用途** — 社媒发布？电商主图？品牌物料？个人收藏？
4. **指定风格** — 摄影写实？插画风格？电影感？极简？
5. **确认比例** — 9:16竖版？16:9横版？1:1方图？

**输出**：参考底部「输出模板 → 模糊需求启发输出」格式，提供 2-3 个方向让用户选择。

### 功能三：二次创作生成

当用户需求明确后，基于 db 中的案例进行二次创作。

**执行步骤**：
1. 读取 `db/<类型名>/prompt.md` 中该类型的类似案例作为参考
2. 根据用户的具体需求（主题、风格、元素、用途）进行二次创作
3. 参考 README 中的避坑指南，确保提示词质量
4. 参考底部「输出模板 → 二次创作输出」格式展示

### 功能四：补充案例（可选）

如遇到 db 中未覆盖的新类型或高质量新案例，可追加写入对应类型的 `prompt.md` 和 `images/` 目录。写入前先确认归属类型是否正确。

## 提示词质量检查清单

生成提示词后，自检以下要点：

- [ ] **类型明确** — 海报/UI/摄影等，类型清晰
- [ ] **主体具体** — 画什么、什么人/物，描述具体
- [ ] **风格锁定** — 插画风/摄影风/电影感等，风格明确
- [ ] **参数完整** — 比例、分辨率、构图等关键参数
- [ ] **避坑到位** — 根据 README 中的避坑指南检查
- [ ] **文字可读** — 如需显示文字，要求文字清晰可读
- [ ] **无歧义** — 关键元素不会被AI误解或自由发挥
